  • IMPORTANT UPDATE:
    • New Users have the "Block & Re-load" Stripping Method chosen as the default. This offers the most privacy benefits.
    • All Users have had the "Skip Known Redirects" feature enabled, though it will only have an affect if you are using the "Block & Re-load" Stripping Method. This adds additional privacy benefits.
    • If you had been using the "Cancel & Reload" method, you have been changed to the "Block and Re-load" Stripping Method.
    If you want the best privacy, I recommend choosing the "Block and Re-load" Stripping Method and enabling the "Skip Known Redirects" checkbox.

    One recent addition to the extension that's quite important is the ability to "Skip Known Redirects." If enabled, this feature will recognize certain links that will pass you through some "intermediate" URL before being redirected to the final URL that you're after, called the "target". An example of this would be Google (Organic) Search result links where you are briefly passed through a Google tracking page before being redirected to the target URL of the search result you clicked. Some affiliate links also do this same type of thing. These links all have some things in common:

    1. The intermediate URL that is visited is not the target URL that you are ultimately going to land on.
    2. The intermediate URL is usually there only to cookie/track you.
    3. The actual target URL is embedded somewhere in the parameters of the intermediate URL, so we can grab it and skip the intermediate step.
